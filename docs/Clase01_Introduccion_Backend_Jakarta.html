DIAPOSITIVA 1 — Modelo Cliente-Servidor

Las aplicaciones web siguen el modelo cliente-servidor. El navegador (cliente) envía peticiones HTTP al servidor, que las procesa y responde con HTML, JSON u otro contenido.

DIAPOSITIVA 2 — HTTP: Protocolo sin estado
📤 Request (Petición)
  • Método: GET, POST, PUT, DELETE
  • URL: /AppVeterinaria01/index.xhtml
  • Headers: Content-Type, Accept
  • Body: datos del formulario (POST)
📥 Response (Respuesta)
  • Status: 200 OK, 404 Not Found, 500 Error
  • Headers: Content-Type: text/html
  • Body: HTML renderizado
  • Stateless: cada petición es independiente

Arquitectura Java para Aplicaciones Web
DIAPOSITIVA 4 — Capas de la aplicación

Una aplicación Jakarta EE bien diseñada se organiza en capas. Cada capa tiene una responsabilidad única y se comunica solo con la capa adyacente.

🎨
Capa Presentación
Páginas XHTML + Facelets — lo que ve el usuario
JSF 4.0
↓ ↑ Eventos y datos
🎮
Capa Controlador (Managed Beans)
@Named @RequestScoped — conecta la vista con el negocio
CDI 4.0
↓ ↑ Llamadas a métodos
⚙️
Capa Negocio (Servicios)
Lógica de negocio, validaciones, transacciones
@Transactional
↓ ↑ Objetos entidad
💾
Capa Datos (DAO / Repository)
EntityManager, JPQL, persist, find, remove
JPA 3.1
↓ ↑ SQL automático
🗄️
Base de Datos
Tablas, registros, relaciones — persistencia real
MySQL 8
✅
Principio clave: Cada capa tiene una sola responsabilidad. La vista no accede a la base de datos. El DAO no conoce el HTML. Esto permite cambiar una capa sin afectar las demás.

Componentes de Jakarta EE 10
DIAPOSITIVA 5 — Tecnologías que usaremos

Jakarta EE 10 es un conjunto de especificaciones. Nosotros usamos el Web Profile (perfil web), que incluye todo lo necesario para aplicaciones web sin la complejidad de la plataforma completa.

Tecnología¿Qué es?¿Para qué la usamos en el curso?
Jakarta Faces 4.0 (JSF) Framework MVC basado en componentes para la capa de presentación Crear las páginas web (.xhtml) con formularios, tablas y navegación
CDI 4.0 Inyección de dependencias y contextos Conectar beans con @Named, @Inject, @RequestScoped
JPA 3.1 Mapeo objeto-relacional (ORM) Mapear clases Java a tablas MySQL con @Entity, @Id, @Column
Bean Validation 3.0 Validaciones declarativas con anotaciones Validar datos con @NotBlank, @Size, @Min, @Email
Jakarta Transactions Manejo de transacciones Asegurar integridad de datos con @Transactional
EclipseLink Implementación de referencia de JPA Motor que ejecuta el SQL real contra MySQL (viene con GlassFish)
⚠️
Nota: Usamos Web Profile, NO Full Platform. No necesitamos EJB, JMS, ni SOAP. El perfil web es más liviano y suficiente para nuestras aplicaciones.

Introducción a JSF (Jakarta Server Faces)
DIAPOSITIVA 7 — ¿Qué es JSF?
🎯 ¿Qué es JSF?
  • Framework MVC basado en componentes
  • Las páginas se escriben en XHTML (no HTML puro)
  • Usa un Faces Servlet que intercepta todas las peticiones
  • Los componentes (h:inputText, h:dataTable) generan HTML automáticamente
  • Se conecta con beans Java mediante Expression Language: #{bean.propiedad}
🔄 ¿Cómo funciona?
  1. El usuario solicita /index.xhtml
  2. GlassFish redirige al Faces Servlet
  3. JSF lee el archivo .xhtml
  4. Ejecuta el ciclo de vida (6 fases)
  5. Genera HTML estándar
  6. Envía la respuesta al navegador
DIAPOSITIVA 8 — Namespaces principales
📋 Namespaces XML en páginas JSF
xmlns:h="jakarta.faces.html"       ← componentes HTML (h:form, h:inputText...)
xmlns:f="jakarta.faces.core"       ← funcionalidades core (f:viewParam, f:ajax...)
xmlns:ui="jakarta.faces.facelets"  ← plantillas (ui:composition, ui:define...)
DIAPOSITIVA 9 — Componentes básicos de JSF
Componente JSFHTML que generaUso típico
<h:form><form>Contenedor de formulario (obligatorio para enviar datos)
<h:inputText><input type="text">Campo de texto ligado a un bean: value="#{bean.campo}"
<h:commandButton><input type="submit">Botón que ejecuta un método: action="#{bean.metodo}"
<h:outputText><span>Mostrar texto dinámico: value="#{bean.resultado}"
<h:dataTable><table>Tabla que itera una lista: value="#{bean.lista}" var="item"
<h:message><span> con errorMuestra mensajes de validación para un campo específico

DIAPOSITIVA 12 — pom.xml generado
📄 pom.xml (fragmento clave)
<project>
    <groupId>com.veterinaria</groupId>
    <artifactId>AppVeterinaria01</artifactId>
    <version>1.0-SNAPSHOT</version>
    <packaging>war</packaging>

    <properties>
        <maven.compiler.release>21</maven.compiler.release>
        <project.build.sourceEncoding>UTF-8</project.build.sourceEncoding>
    </properties>

    <dependencies>
        <dependency>
            <groupId>jakarta.platform</groupId>
            <artifactId>jakarta.jakartaee-web-api</artifactId>
            <version>10.0.0</version>
            <scope>provided</scope>  <!-- GlassFish ya lo tiene -->
        </dependency>
    </dependencies>
</project>pom.xml
💡
scope=provided: Significa que GlassFish ya incluye estas librerías. Maven las usa solo para compilar, pero NO las empaqueta en el .war. Esto evita conflictos de versiones.

Archivos de configuración
DIAPOSITIVA 14 — web.xml
📄 WEB-INF/web.xml
<?xml version="1.0" encoding="UTF-8"?>
<web-app xmlns="https://jakarta.ee/xml/ns/jakartaee"
         version="6.0">

    <!-- Faces Servlet: intercepta peticiones .xhtml -->
    <servlet>
        <servlet-name>Faces Servlet</servlet-name>
        <servlet-class>jakarta.faces.webapp.FacesServlet</servlet-class>
        <load-on-startup>1</load-on-startup>
    </servlet>

    <servlet-mapping>
        <servlet-name>Faces Servlet</servlet-name>
        <url-pattern>*.xhtml</url-pattern>
    </servlet-mapping>

    <!-- Página de inicio -->
    <welcome-file-list>
        <welcome-file>index.xhtml</welcome-file>
    </welcome-file-list>

</web-app>web.xml
DIAPOSITIVA 15 — beans.xml
📄 WEB-INF/beans.xml
<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="https://jakarta.ee/xml/ns/jakartaee"
       version="4.0"
       bean-discovery-mode="annotated">
</beans>beans.xml
⚙️ web.xml — ¿Para qué sirve?
  • Registra el Faces Servlet
  • Mapea las URLs *.xhtml al servlet JSF
  • Define la página de bienvenida
  • Sin este archivo, JSF no procesa las páginas
📦 beans.xml — ¿Para qué sirve?
  • Activa el sistema CDI (inyección de dependencias)
  • bean-discovery-mode="annotated": solo detecta clases con anotaciones CDI
  • Sin este archivo, @Named y @Inject no funcionan
  • En Jakarta EE 10, es opcional si se usa modo "annotated"

DIAPOSITIVA 17 — SaludoBean.java
📄 controlador/SaludoBean.java
package com.veterinaria.controlador;

import jakarta.enterprise.context.RequestScoped;
import jakarta.inject.Named;

@Named           // hace el bean accesible desde JSF: #{saludoBean}
@RequestScoped   // vive durante una petición HTTP
public class SaludoBean {

    private String nombre;
    private String mensaje;

    // Método que ejecuta el botón
    public void saludar() {
        this.mensaje = "¡Hola " + nombre + "! Bienvenido al sistema.";
    }

    // Getters y Setters
    public String getNombre()              { return nombre; }
    public void   setNombre(String n)      { this.nombre = n; }
    public String getMensaje()             { return mensaje; }
    public void   setMensaje(String m)     { this.mensaje = m; }
}SaludoBean.java
✅
¿Cómo se conectan? La página usa #{saludoBean.nombre} que busca un bean llamado saludoBean (primera letra minúscula del nombre de la clase). @Named registra la clase. @RequestScoped define su ciclo de vida.

DIAPOSITIVA 20 — Errores comunes y soluciones
⚠️ Error🔍 Causa probable✅ Solución
404 Not Found La URL no coincide con el contexto de la app Verificar que la URL sea /AppVeterinaria01/index.xhtml
500 Internal Server Error Error en el código Java o configuración Revisar la consola de GlassFish (pestaña Output en NetBeans)
Página en blanco Faces Servlet no está mapeado Verificar web.xml: url-pattern debe ser *.xhtml
Target Unreachable: 'saludoBean' CDI no detecta el bean Verificar que exista beans.xml y que la clase tenga @Named
GlassFish no arranca Puerto 8080 ocupado por otro proceso Cerrar el proceso que usa el puerto o cambiar el puerto en GlassFish
ClassNotFoundException Dependencia faltante o scope incorrecto Verificar pom.xml: jakarta.jakartaee-web-api con scope provided
