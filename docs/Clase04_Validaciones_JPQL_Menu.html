Bean Validation en la Entidad
DIAPOSITIVA 7 — Anotaciones Bean Validation

Bean Validation 3.0 permite declarar restricciones directamente en los campos de la entidad. JPA las valida automáticamente antes de persist() y merge().

Anotación¿Qué valida?Aplica a
@NotNullNo puede ser nullCualquier tipo
@NotBlankNo null, no vacío, no solo espaciosSolo String
@NotEmptyNo null y no vacío (length > 0)String, Collection, Map
@Size(min, max)Longitud entre min y maxString, Collection
@EmailFormato de email válidoString
@PastFecha anterior a hoyLocalDate, Date

DIAPOSITIVA 8 — @NotBlank vs @NotNull vs @NotEmpty
Valor del campo@NotNull@NotEmpty@NotBlank
null❌ Falla❌ Falla❌ Falla
"" (vacío)✅ Pasa❌ Falla❌ Falla
" " (espacios)✅ Pasa✅ Pasa❌ Falla
"abc"✅ Pasa✅ Pasa✅ Pasa
💡
Recomendación: Para campos String obligatorios, usar siempre @NotBlank porque es la más estricta: rechaza null, vacío Y solo espacios.
DIAPOSITIVA 9 — Cliente.java con Bean Validation completo
📄 modelo/Cliente.java — Con validaciones
package com.veterinaria.modelo;

import jakarta.persistence.*;
import jakarta.validation.constraints.*;
import java.time.LocalDate;

@Entity
@Table(name = "clientes")
public class Cliente {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private int id;

    @NotBlank(message = "El documento es obligatorio")
    @Size(min = 7, max = 20, message = "Documento: 7 a 20 caracteres")
    @Column(nullable = false, unique = true, length = 20)
    private String documento;

    @NotBlank(message = "El nombre es obligatorio")
    @Size(min = 2, max = 100, message = "Nombre: 2 a 100 caracteres")
    @Column(nullable = false, length = 100)
    private String nombre;

    @NotBlank(message = "El correo es obligatorio")
    @Email(message = "El formato de correo no es válido")
    @Column(unique = true, length = 150)
    private String correo;

    @Past(message = "La fecha debe ser anterior a hoy")
    @Column(name = "fecha_nacimiento")
    private LocalDate fechaNacimiento;

    // Constructor vacío
    public Cliente() {}

    // Getters y Setters
    public int getId() { return id; }
    public void setId(int id) { this.id = id; }
    public String getDocumento() { return documento; }
    public void setDocumento(String d) { this.documento = d; }
    public String getNombre() { return nombre; }
    public void setNombre(String n) { this.nombre = n; }
    public String getCorreo() { return correo; }
    public void setCorreo(String c) { this.correo = c; }
    public LocalDate getFechaNacimiento() { return fechaNacimiento; }
    public void setFechaNacimiento(LocalDate f) { this.fechaNacimiento = f; }
}Cliente.java
✅
Integración automática: JSF valida en la fase 3 (Process Validations). Si pasa esa fase y llega a JPA, Bean Validation se ejecuta automáticamente antes de em.persist(). Doble protección sin código extra.

Validaciones de Negocio en el Servicio
DIAPOSITIVA 10 — Lo que Bean Validation NO puede hacer
✅ Bean Validation PUEDE
  • Validar formato (email, regex)
  • Validar largo (min/max)
  • Validar obligatoriedad (not null/blank)
  • Validar rango de fechas (past/future)
❌ Bean Validation NO PUEDE
  • Verificar unicidad en la BD
  • Comparar con otros registros existentes
  • Reglas que requieran consultar datos
  • Lógica de negocio compleja
DIAPOSITIVA 11 — ClienteServicio con validación de unicidad
📄 servicio/ClienteServicio.java
package com.veterinaria.servicio;

import com.veterinaria.dao.ClienteDAO;
import com.veterinaria.modelo.Cliente;
import jakarta.enterprise.context.ApplicationScoped;
import jakarta.inject.Inject;
import jakarta.transaction.Transactional;
import java.util.List;
import java.util.Optional;

@ApplicationScoped
public class ClienteServicio {

    @Inject
    private ClienteDAO clienteDAO;

    @Transactional
    public void guardar(Cliente cliente) {
        // Validar unicidad de documento
        Optional<Cliente> porDoc = clienteDAO.buscarPorDocumento(cliente.getDocumento());
        if (porDoc.isPresent() && porDoc.get().getId() != cliente.getId()) {
            throw new IllegalArgumentException(
                "Ya existe un cliente con documento: " + cliente.getDocumento());
        }

        // Validar unicidad de correo
        Optional<Cliente> porEmail = clienteDAO.buscarPorEmail(cliente.getCorreo());
        if (porEmail.isPresent() && porEmail.get().getId() != cliente.getId()) {
            throw new IllegalArgumentException(
                "Ya existe un cliente con correo: " + cliente.getCorreo());
        }

        // Si pasa las validaciones, guardar
        if (cliente.getId() == 0) {
            clienteDAO.crear(cliente);
        } else {
            clienteDAO.actualizar(cliente);
        }
    }

    public List<Cliente> listarTodos() {
        return clienteDAO.listarTodos();
    }

    public List<Cliente> buscar(String texto) {
        return clienteDAO.buscarPorTexto(texto);
    }

    @Transactional
    public void eliminar(int id) {
        clienteDAO.eliminar(id);
    }
}ClienteServicio.java

JPQL — Jakarta Persistence Query Language
DIAPOSITIVA 13 — SQL vs JPQL

JPQL es el lenguaje de consultas de JPA. Se parece a SQL pero trabaja sobre entidades y atributos Java, no sobre tablas y columnas.

SQL (tablas/columnas)JPQL (entidades/atributos)
SELECT * FROM clientesSELECT c FROM Cliente c
SELECT * FROM clientes WHERE documento = ?SELECT c FROM Cliente c WHERE c.documento = :doc
SELECT * FROM clientes WHERE correo = ?SELECT c FROM Cliente c WHERE c.correo = :correo
SELECT * FROM clientes WHERE nombre LIKE '%texto%'SELECT c FROM Cliente c WHERE LOWER(c.nombre) LIKE :texto
DELETE FROM clientes WHERE id = ?DELETE FROM Cliente c WHERE c.id = :id
💡
Diferencia clave: En JPQL escribimos Cliente (nombre de la clase Java), no clientes (nombre de la tabla). Usamos c.nombre (atributo Java), no nombre (columna SQL).

DIAPOSITIVA 15 — Uso en el DAO con TypedQuery
📄 dao/ClienteDAO.java — Métodos JPQL
@ApplicationScoped
public class ClienteDAO extends GenericDAO<Cliente, Integer> {

    public ClienteDAO() {
        super(Cliente.class);
    }

    public Optional<Cliente> buscarPorDocumento(String documento) {
        TypedQuery<Cliente> query = em.createNamedQuery(
            "Cliente.findByDocumento", Cliente.class);
        query.setParameter("documento", documento);
        List<Cliente> resultados = query.getResultList();
        return resultados.isEmpty() ? Optional.empty()
                                    : Optional.of(resultados.get(0));
    }

    public Optional<Cliente> buscarPorEmail(String correo) {
        TypedQuery<Cliente> query = em.createNamedQuery(
            "Cliente.findByEmail", Cliente.class);
        query.setParameter("correo", correo);
        List<Cliente> resultados = query.getResultList();
        return resultados.isEmpty() ? Optional.empty()
                                    : Optional.of(resultados.get(0));
    }

    public List<Cliente> buscarPorTexto(String texto) {
        TypedQuery<Cliente> query = em.createNamedQuery(
            "Cliente.buscarPorTexto", Cliente.class);
        query.setParameter("texto", "%" + texto.toLowerCase() + "%");
        return query.getResultList();
    }
}ClienteDAO.java
✅
TypedQuery: A diferencia de Query, el TypedQuery<Cliente> devuelve directamente una lista tipada sin necesidad de casting. Siempre preferir TypedQuery.

Plantilla Facelets
DIAPOSITIVA 16 — ¿Qué es una plantilla?
📋 ¿Qué es?
  • Un layout compartido para todas las páginas
  • Define zonas reemplazables con ui:insert
  • Evita repetir header, menú y footer en cada página
  • Cada página hija usa ui:composition + ui:define
🔄 Flujo
  1. plantilla.xhtml define la estructura general
  2. Usa ui:insert name="titulo" y ui:insert name="contenido"
  3. Las páginas hijas usan ui:composition template="/plantilla.xhtml"
  4. Cada hija define ui:define name="contenido"

package com.veterinaria.controlador;

import com.veterinaria.modelo.Cliente;
import com.veterinaria.servicio.ClienteServicio;
import jakarta.annotation.PostConstruct;
import jakarta.faces.application.FacesMessage;
import jakarta.faces.context.FacesContext;
import jakarta.faces.view.ViewScoped;
import jakarta.inject.Inject;
import jakarta.inject.Named;
import java.io.Serializable;
import java.util.List;

@Named
@ViewScoped
public class ClienteBean implements Serializable {

    @Inject
    private ClienteServicio clienteServicio;

    private Cliente cliente = new Cliente();
    private List<Cliente> clientes;
    private String textoBusqueda;

    @PostConstruct
    public void init() { listar(); }

    public void listar() {
        clientes = clienteServicio.listarTodos();
        textoBusqueda = null;
    }

    public void buscar() {
        if (textoBusqueda != null && !textoBusqueda.isBlank()) {
            clientes = clienteServicio.buscar(textoBusqueda);
        } else {
            listar();
        }
    }

    public void guardar() {
        try {
            clienteServicio.guardar(cliente);
            FacesContext.getCurrentInstance().addMessage(null,
                new FacesMessage(FacesMessage.SEVERITY_INFO,
                    "Cliente guardado correctamente", null));
            cliente = new Cliente();
            listar();
        } catch (IllegalArgumentException e) {
            FacesContext.getCurrentInstance().addMessage(null,
                new FacesMessage(FacesMessage.SEVERITY_ERROR,
                    e.getMessage(), null));
        }
    }

    public void editar(Cliente c) { this.cliente = c; }

    public void eliminar(Cliente c) {
        clienteServicio.eliminar(c.getId());
        listar();
    }

    public void cancelar() { cliente = new Cliente(); }

    // Getters y Setters
    public Cliente getCliente() { return cliente; }
    public void setCliente(Cliente c) { this.cliente = c; }
    public List<Cliente> getClientes() { return clientes; }
    public String getTextoBusqueda() { return textoBusqueda; }
    public void setTextoBusqueda(String t) { this.textoBusqueda = t; }
}ClienteBean.java
