El problema de la persistencia
DIAPOSITIVA 1 — Antes vs Después
❌ ANTES — Clase 02 (ArrayList en Bean)
  • Datos guardados en ArrayList<Cliente>
  • Viven en la memoria RAM del servidor
  • Al reiniciar GlassFish: ¡datos perdidos!
  • No soporta múltiples usuarios concurrentes
  • No hay búsquedas eficientes
✅ DESPUÉS — Clase 03 (JPA + MySQL)
  • Datos guardados en tablas MySQL
  • Persisten en el disco duro
  • Reiniciar GlassFish: ¡datos intactos!
  • Soporte completo para concurrencia
  • Búsquedas con SQL/JPQL optimizado
DIAPOSITIVA 2 — ¿Qué es ORM?

ORM (Object-Relational Mapping) es una técnica que mapea automáticamente objetos Java a tablas de base de datos. No escribimos SQL manual: JPA lo genera por nosotros.

DIAPOSITIVA 8 — Archivo completo

Este archivo le dice a JPA cómo conectarse a la base de datos y cómo comportarse. Se ubica en src/main/resources/META-INF/persistence.xml.

📄 META-INF/persistence.xml
<?xml version="1.0" encoding="UTF-8"?>
<persistence xmlns="https://jakarta.ee/xml/ns/persistence"
             version="3.0">

    <persistence-unit name="VeterinariaPU"
                      transaction-type="JTA">

        <!-- DataSource configurado en GlassFish -->
        <jta-data-source>jdbc/VeterinariaDB</jta-data-source>

        <!-- Propiedades de EclipseLink -->
        <properties>
            <!-- Mostrar SQL generado en consola -->

                      value="FINE"/>

            <!-- Mostrar parámetros de las queries -->
            <property name="eclipselink.logging.parameters"
                      value="true"/>

            <!-- NO crear/modificar tablas automáticamente -->
            <property name="eclipselink.ddl-generation"
                      value="none"/>

Entidad Cliente con @Entity
DIAPOSITIVA 10 — Cliente.java completo

Una entidad JPA es una clase Java normal (POJO) decorada con anotaciones que indican cómo se mapea a una tabla de la base de datos.

📄 modelo/Cliente.java
package com.veterinaria.modelo;

import jakarta.persistence.*;
import java.io.Serializable;

@Entity                          // Marca la clase como entidad JPA
@Table(name = "cliente")         // Nombre de la tabla en MySQL
@NamedQuery(
    name = "Cliente.findAll",
    query = "SELECT c FROM Cliente c ORDER BY c.nombre"
)
public class Cliente implements Serializable {

    @Id                                      // Clave primaria
    @GeneratedValue(strategy = GenerationType.IDENTITY)  // AUTO_INCREMENT
    private int id;

    @Column(nullable = false, unique = true, length = 20)
    private String documento;

    @Column(nullable = false, length = 100)
    private String nombre;

    // Constructor vacío (requerido por JPA)
    public Cliente() {}

    // Constructor con parámetros
    public Cliente(String documento, String nombre) {
        this.documento = documento;
        this.nombre = nombre;
    }

    // Getters y Setters
    public int getId()                    { return id; }
    public void setId(int id)             { this.id = id; }
    public String getDocumento()          { return documento; }
    public void setDocumento(String d)    { this.documento = d; }
    public String getNombre()             { return nombre; }
    public void setNombre(String n)       { this.nombre = n; }
}Cliente.java
DIAPOSITIVA 11 — Explicación de anotaciones
AnotaciónUbicaciónFunción
@EntityClaseIndica que esta clase se persiste en una tabla
@Table(name="cliente")ClaseEspecifica el nombre de la tabla (si difiere del nombre de la clase)
@IdCampoMarca el campo como clave primaria
@GeneratedValue(IDENTITY)CampoEl valor lo genera MySQL con AUTO_INCREMENT
@Column(nullable, unique, length)CampoRestricciones de la columna: NOT NULL, UNIQUE, tamaño
@NamedQueryClaseQuery JPQL predefinida reutilizable (compilada al iniciar)
💡
¿Por qué int y no Long? Nuestra tabla usa INT AUTO_INCREMENT. Para mantener coherencia, usamos int (primitivo) en Java. Si la tabla usara BIGINT, usaríamos long o Long.

GenericDAO abstracto
DIAPOSITIVA 12 — ¿Qué es el patrón DAO?

El patrón DAO (Data Access Object) separa la lógica de acceso a datos del resto de la aplicación. Usamos genéricos para no repetir código en cada entidad.

🧩 ¿Por qué Genéricos?
  • Un solo DAO base para todas las entidades
  • GenericDAO<T, ID> donde T=Entidad, ID=tipo de clave
  • ClienteDAO extiende GenericDAO<Cliente, Integer>
  • MascotaDAO extiende GenericDAO<Mascota, Integer>
  • Evita duplicar persist, merge, remove, find en cada DAO
💾 EntityManager — Métodos clave
  • persist(entity) — INSERT (guardar nuevo)
  • merge(entity) — UPDATE (actualizar existente)
  • remove(entity) — DELETE (eliminar)
  • find(Class, id) — SELECT por PK (buscar por id)
  • createQuery(jpql) — SELECT personalizado
DIAPOSITIVA 13 — GenericDAO.java completo
📄 dao/GenericDAO.java
package com.veterinaria.dao;

import jakarta.persistence.EntityManager;
import jakarta.persistence.PersistenceContext;
import jakarta.transaction.Transactional;
import java.util.List;

public abstract class GenericDAO<T, ID> {

    @PersistenceContext(unitName = "VeterinariaPU")
    protected EntityManager em;

    private Class<T> entityClass;

    // El hijo debe pasar la clase en su constructor
    public GenericDAO(Class<T> entityClass) {
        this.entityClass = entityClass;
    }

    @Transactional
    public void guardar(T entidad) {
        em.persist(entidad);           // INSERT INTO ...
    }

    @Transactional
    public T actualizar(T entidad) {
        return em.merge(entidad);      // UPDATE ... SET ...
    }

    @Transactional
    public void eliminar(ID id) {
        T entidad = em.find(entityClass, id);  // Buscar primero
        if (entidad != null) {
            em.remove(entidad);        // DELETE FROM ...
        }
    }

    public T buscarPorId(ID id) {
        return em.find(entityClass, id);  // SELECT ... WHERE id = ?
    }

    public List<T> listarTodos() {
        return em.createQuery(
            "SELECT e FROM " + entityClass.getSimpleName() + " e",
            entityClass
        ).getResultList();
    }
}GenericDAO.java
✅
@PersistenceContext: GlassFish inyecta automáticamente el EntityManager. No necesitamos crear ni cerrar conexiones manualmente. @Transactional: GlassFish abre y cierra la transacción automáticamente (commit si todo sale bien, rollback si hay error).

ClienteDAO
DIAPOSITIVA 14 — ClienteDAO.java completo

El ClienteDAO extiende GenericDAO y agrega métodos específicos de la entidad Cliente.

📄 dao/ClienteDAO.java
package com.veterinaria.dao;

import com.veterinaria.modelo.Cliente;
import jakarta.enterprise.context.ApplicationScoped;
import java.util.List;

@ApplicationScoped
public class ClienteDAO extends GenericDAO<Cliente, Integer> {

    public ClienteDAO() {
        super(Cliente.class);
    }

    // Usa la @NamedQuery definida en la entidad
    @Override
    public List<Cliente> listarTodos() {
        return em.createNamedQuery("Cliente.findAll", Cliente.class)
                  .getResultList();
    }

    // Método específico: buscar por documento
    public Cliente buscarPorDocumento(String documento) {
        List<Cliente> resultados = em.createQuery(
            "SELECT c FROM Cliente c WHERE c.documento = :doc",
            Cliente.class
        ).setParameter("doc", documento)
         .getResultList();

        return resultados.isEmpty() ? null : resultados.get(0);
    }
}ClienteDAO.java
📋 @ApplicationScoped
Una sola instancia para toda la aplicación. Los DAOs no guardan estado del usuario, así que una instancia compartida es eficiente.
🔄 Herencia Genérica
GenericDAO<Cliente, Integer> significa: entidad Cliente, clave primaria tipo Integer. Hereda guardar, actualizar, eliminar, buscarPorId.

ClienteServicio
DIAPOSITIVA 15 — Capa de servicio

La capa de servicio contiene la lógica de negocio. Valida datos antes de llamar al DAO. El Bean nunca accede al DAO directamente.

📄 servicio/ClienteServicio.java
package com.veterinaria.servicio;

import com.veterinaria.dao.ClienteDAO;
import com.veterinaria.modelo.Cliente;
import jakarta.enterprise.context.ApplicationScoped;
import jakarta.inject.Inject;
import jakarta.transaction.Transactional;
import java.util.List;

@ApplicationScoped
public class ClienteServicio {

    @Inject
    private ClienteDAO clienteDAO;

    @Transactional
    public void guardar(Cliente cliente) {
        // Validar que no exista otro con el mismo documento
        Cliente existente = clienteDAO.buscarPorDocumento(
            cliente.getDocumento()
        );
        if (existente != null && existente.getId() != cliente.getId()) {
            throw new RuntimeException(
                "Ya existe un cliente con documento: "
                + cliente.getDocumento()
            );
        }

        if (cliente.getId() == 0) {
            clienteDAO.guardar(cliente);       // INSERT (nuevo)
        } else {
            clienteDAO.actualizar(cliente);    // UPDATE (existente)
        }
    }

    public List<Cliente> listarTodos() {
        return clienteDAO.listarTodos();
    }

    public Cliente buscarPorId(int id) {
        return clienteDAO.buscarPorId(id);
    }

    @Transactional
    public void eliminar(int id) {
        clienteDAO.eliminar(id);
    }
}ClienteServicio.java
💡
¿Por qué @Transactional en el Servicio? La transacción debe abarcar toda la operación de negocio. Si la validación falla después de una escritura, se hace rollback completo. El servicio es el dueño de la transacción.

DIAPOSITIVA 17 — ClienteBean.java completo
📄 controlador/ClienteBean.java
package com.veterinaria.controlador;

import com.veterinaria.modelo.Cliente;
import com.veterinaria.servicio.ClienteServicio;
import jakarta.annotation.PostConstruct;
import jakarta.faces.application.FacesMessage;
import jakarta.faces.context.FacesContext;
import jakarta.faces.view.ViewScoped;
import jakarta.inject.Inject;
import jakarta.inject.Named;
import java.io.Serializable;
import java.util.List;

@Named                           // Accesible desde JSF: #{clienteBean}
@ViewScoped                      // Vive mientras el usuario esté en la página
public class ClienteBean implements Serializable {

    @Inject
    private ClienteServicio servicio;

    private Cliente cliente = new Cliente();
    private List<Cliente> clientes;

    @PostConstruct
    public void init() {
        clientes = servicio.listarTodos();  // Carga desde MySQL
    }

    public void guardar() {
        try {
            servicio.guardar(cliente);
            cliente = new Cliente();          // Limpiar formulario
            clientes = servicio.listarTodos(); // Refrescar lista
            addMessage(FacesMessage.SEVERITY_INFO,
                "Éxito", "Cliente guardado correctamente");
        } catch (Exception e) {
            addMessage(FacesMessage.SEVERITY_ERROR,
                "Error", e.getMessage());
        }
    }

    public void eliminar(Cliente c) {
        try {
            servicio.eliminar(c.getId());
            clientes = servicio.listarTodos();
            addMessage(FacesMessage.SEVERITY_INFO,
                "Éxito", "Cliente eliminado");
        } catch (Exception e) {
            addMessage(FacesMessage.SEVERITY_ERROR,
                "Error", e.getMessage());
        }
    }

    public void prepararEdicion(Cliente c) {
        this.cliente = c;  // Carga el cliente en el formulario
    }

    public void cancelar() {
        this.cliente = new Cliente();  // Limpia el formulario
    }

    private void addMessage(FacesMessage.Severity sev,
                            String titulo, String detalle) {
        FacesContext.getCurrentInstance().addMessage(null,
            new FacesMessage(sev, titulo, detalle));
    }

    // Getters y Setters
    public Cliente getCliente()              { return cliente; }
    public void setCliente(Cliente c)       { this.cliente = c; }
    public List<Cliente> getClientes()     { return clientes; }
}ClienteBean.java
⚠️
@ViewScoped: El bean vive mientras el usuario esté en la misma página. Esto permite editar un cliente sin perder el estado del formulario. Requiere implements Serializable.

DIAPOSITIVA 24 — Actividad práctica
🏆 Actividad práctica — CRUD Cliente con JPA
1

Crear la base de datos

Ejecutar el script SQL para crear veterinariadb y la tabla cliente en MySQL.

2

Configurar DataSource en GlassFish

Crear Connection Pool (VeterinariaPool) y JDBC Resource (jdbc/VeterinariaDB). Hacer Ping para verificar.

3

Crear proyecto AppVeterinaria03

Maven Web Application, Jakarta EE 10, GlassFish 7. Crear los paquetes: modelo, dao, servicio, controlador.

4

Crear las clases Java

Crear en orden: Cliente.java → GenericDAO.java → ClienteDAO.java → ClienteServicio.java → ClienteBean.java

5

Configurar persistence.xml y crear index.xhtml

Copiar persistence.xml en META-INF. Crear el index.xhtml con formulario y tabla CRUD.

6

Probar el CRUD completo

Desplegar en GlassFish. Verificar: agregar clientes, editar, eliminar. Reiniciar GlassFish y verificar que los datos persisten.

📌
Entregable: Proyecto AppVeterinaria03 funcionando con CRUD de clientes contra MySQL. Los datos deben persistir después de reiniciar GlassFish.
