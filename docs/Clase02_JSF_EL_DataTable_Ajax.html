Expression Language (EL)
DIAPOSITIVA 1 — ¿Qué es Expression Language?

Expression Language (EL) es el mecanismo que conecta las páginas XHTML con los beans Java. Permite leer propiedades, ejecutar métodos y evaluar expresiones directamente en la vista.

🔗 Value Binding (lectura/escritura)

#{bean.propiedad}

  • Lee el valor: llama a getPropiedad()
  • Escribe el valor: llama a setPropiedad(valor)
  • Ejemplo: #{clienteBean.nombre}
🎯 Action Binding (ejecución)

#{bean.metodo()}

  • Ejecuta un método del bean
  • Usado en action y actionListener
  • Ejemplo: #{clienteBean.agregar()}
DIAPOSITIVA 2 — #{} vs ${}
⚠️ Diferencia entre #{} y ${}
SintaxisTipoEvaluación¿Usar en JSF?
#{expresion}Deferred (diferida)Se evalúa cuando JSF lo necesita (get/set)SÍ — siempre
${expresion}Immediate (inmediata)Se evalúa una sola vez al renderizarNO — solo lectura
⚠️
Regla: En JSF siempre usamos #{}. La sintaxis ${} es de JSP/JSTL y en JSF solo funciona para lectura, no para escribir valores en el bean.

Managed Bean con CDI
DIAPOSITIVA 5 — Anotaciones CDI principales
@Named
  • Hace que el bean sea accesible desde EL
  • El nombre por defecto es la clase con minúscula inicial
  • @Named("clienteBean") → #{clienteBean.xxx}
  • Pertenece a jakarta.inject.Named
@ViewScoped
  • El bean vive mientras el usuario permanezca en la misma página
  • Sobrevive a peticiones Ajax y postbacks
  • Se destruye al navegar a otra página
  • La clase debe implementar Serializable
  • Pertenece a jakarta.faces.view.ViewScoped
@RequestScoped
  • El bean vive solo durante UNA petición HTTP
  • Se crea y destruye en cada request
  • Útil para formularios simples sin tabla
  • Pertenece a jakarta.enterprise.context.RequestScoped
@PostConstruct
  • Método que se ejecuta después de crear el bean
  • Ideal para cargar datos iniciales
  • Se ejecuta UNA vez por instancia del bean
  • Pertenece a jakarta.annotation.PostConstruct
💡
¿Cuándo usar @ViewScoped? Siempre que la página tenga una tabla con datos y formulario de edición. Si usáramos @RequestScoped, la lista se perdería entre cada click de botón.

package com.veterinaria.controlador;

import com.veterinaria.modelo.Cliente;
import jakarta.annotation.PostConstruct;
import jakarta.faces.application.FacesMessage;
import jakarta.faces.context.FacesContext;
import jakarta.faces.view.ViewScoped;
import jakarta.inject.Named;
import java.io.Serializable;
import java.util.ArrayList;
import java.util.List;

@Named("clienteBean")
@ViewScoped
public class ClienteBean implements Serializable {

    private List<Cliente> clientes = new ArrayList<>();
    private Cliente clienteSeleccionado = new Cliente();
    private int proximoId = 1;

    @PostConstruct
    public void init() {
        // Datos de ejemplo en memoria
        clientes.add(new Cliente(1, "30123456", "María Gómez"));
        clientes.add(new Cliente(2, "28987654", "Carlos López"));
        proximoId = 3;
    }

    public void agregar() {
        clienteSeleccionado.setId(proximoId++);
        clientes.add(clienteSeleccionado);
        addMsg(FacesMessage.SEVERITY_INFO, "Cliente agregado correctamente");
        clienteSeleccionado = new Cliente();
    }

    public void eliminar(Cliente c) {
        clientes.remove(c);
        addMsg(FacesMessage.SEVERITY_WARN, "Cliente eliminado: " + c.getNombre());
    }

    public void prepararEdicion(Cliente c) {
        clienteSeleccionado = c;
    }

    private void addMsg(FacesMessage.Severity severity, String texto) {
        FacesContext.getCurrentInstance()
            .addMessage(null, new FacesMessage(severity, texto, null));
    }

    // Getters y Setters
    public List<Cliente> getClientes()                     { return clientes; }
    public void setClientes(List<Cliente> clientes)        { this.clientes = clientes; }
    public Cliente getClienteSeleccionado()                 { return clienteSeleccionado; }
    public void setClienteSeleccionado(Cliente c)         { this.clienteSeleccionado = c; }
}ClienteBean.java

FacesMessage — Feedback al usuario
DIAPOSITIVA 11 — Niveles de severidad

JSF provee un sistema de mensajes integrado. Desde el bean podemos enviar mensajes que se muestran automáticamente en la vista con componentes específicos.

✅ SEVERITY_INFO

Éxito / Confirmación

Operación completada correctamente.

"Cliente agregado"

⚠️ SEVERITY_WARN

Advertencia / Error de negocio

Algo no está bien pero no es crítico.

"Documento duplicado"

❌ SEVERITY_ERROR

Error técnico / grave

Algo falló y la operación no se completó.

"Error al guardar"

📄 Método helper addMsg()
// Método auxiliar para agregar mensajes fácilmente
private void addMsg(FacesMessage.Severity severity, String texto) {
    FacesContext.getCurrentInstance()
        .addMessage(null, new FacesMessage(severity, texto, null));
}

// Uso en los métodos del bean:
public void agregar() {
    if (documentoExiste(clienteSeleccionado.getDocumento())) {
        addMsg(FacesMessage.SEVERITY_WARN, "El documento ya existe");
        return;
    }
    clienteSeleccionado.setId(proximoId++);
    clientes.add(clienteSeleccionado);
    addMsg(FacesMessage.SEVERITY_INFO, "Cliente agregado correctamente");
    clienteSeleccionado = new Cliente();  // limpiar formulario
}

public void eliminar(Cliente c) {
    clientes.remove(c);
    addMsg(FacesMessage.SEVERITY_WARN, "Cliente eliminado: " + c.getNombre());
}FacesMessage
⚠️
Primer parámetro null: Cuando pasamos null como clientId en addMessage(null, ...), el mensaje es global y se muestra en <h:messages globalOnly="true">. Si pasáramos un id de componente, se mostraría en el <h:message for="id"> correspondiente.

f:ajax — Actualizaciones parciales
DIAPOSITIVA 14 — ¿Qué es Ajax en JSF?
⚡ Ajax = Actualizar parte de la página sin recargar

Sin Ajax, cada botón recarga TODA la página. Con <f:ajax> podemos:

  • Enviar solo parte del formulario al servidor
  • Actualizar solo la sección de la página que cambió
  • Mantener la posición de scroll del usuario
  • Experiencia más fluida, similar a aplicaciones de escritorio
DIAPOSITIVA 15 — Patrón más común
📄 f:ajax execute + render
<!-- Patrón más común: enviar y actualizar todo el formulario -->
<h:commandButton value="Guardar"
    action="#{clienteBean.agregar()}">
    <f:ajax execute="@form" render="@form" />
</h:commandButton>

<!-- Actualizar otro formulario (el listado) -->
<h:commandButton value="Guardar"
    action="#{clienteBean.agregar()}">
    <f:ajax execute="@form" render="@form :frmListado" />
</h:commandButton>

<!-- Búsqueda en tiempo real con evento keyup -->
<h:inputText value="#{clienteBean.filtro}"
    pt:placeholder="Buscar cliente...">
    <f:ajax event="keyup"
        execute="@this"
        render="tablaClientes"
        listener="#{clienteBean.filtrar()}" />
</h:inputText>f:ajax
💡
xmlns:pt — Para usar atributos HTML como placeholder en componentes JSF, necesitamos el namespace passthrough: xmlns:pt="jakarta.faces.passthrough" en el html tag.
DIAPOSITIVA 16 — execute vs render
Atributo¿Qué hace?Valores comunesDirección
executeQué componentes se ENVÍAN al servidor@form, @this, @all, id📤 Cliente → Servidor
renderQué componentes se ACTUALIZAN en pantalla@form, @this, @all, :id📥 Servidor → Cliente
📤 Valores de execute
  • @form — todos los campos del formulario actual
  • @this — solo el componente que disparó el evento
  • @all — toda la página (raro, evitar)
  • campo1 campo2 — IDs específicos separados por espacio
📥 Valores de render
  • @form — re-renderizar el formulario actual
  • @this — solo el componente actual
  • @none — no actualizar nada visible
  • :frmListado — ID absoluto (dos puntos = raíz)

📄 Método filtrar() en el Bean
private String filtro = "";
private List<Cliente> clientesFiltrados = new ArrayList<>();

public void filtrar() {
    if (filtro == null || filtro.isBlank()) {
        clientesFiltrados = new ArrayList<>(clientes);
    } else {
        clientesFiltrados = clientes.stream()
            .filter(c -> c.getNombre().toLowerCase()
                .contains(filtro.toLowerCase()))
            .toList();
    }
}filtrar()

Ciclo de Vida de JSF (6 fases)
DIAPOSITIVA 18 — Las 6 fases del ciclo de vida

Cada vez que el usuario envía un formulario JSF, el Faces Servlet ejecuta un ciclo de 6 fases en orden. Entender este ciclo es clave para depurar errores.

FaseNombre¿Qué hace?
1Restore ViewReconstruye el árbol de componentes de la página
2Apply Request ValuesToma los valores enviados por el navegador y los asigna a los componentes
3Process ValidationsEjecuta validaciones (required, @Size, convertidores)
4Update Model ValuesCopia los valores validados a las propiedades del bean (setters)
5Invoke ApplicationEjecuta el método action del botón (ej: agregar())
6Render ResponseGenera el HTML de respuesta y lo envía al navegador

DIAPOSITIVA 20 — immediate=true y validación fallida
❌ Cuando la validación falla
  • Si un campo con required="true" está vacío, la validación falla en fase 3
  • JSF salta directamente a la fase 6 (Render Response)
  • Las fases 4 y 5 NO se ejecutan
  • El método action del botón nunca se llama
  • Se muestran los mensajes de error en <h:message>
⚡ immediate="true"
  • Salta las fases 2, 3 y 4
  • Ejecuta la acción inmediatamente después de fase 1
  • Útil para botones Cancelar que no necesitan validar
  • Ejemplo: <h:commandButton value="Cancelar" immediate="true" action="#{bean.cancelar()}" />
✅
Tip práctico: Si tu botón "Guardar" no ejecuta el método, probablemente hay un error de validación. Agrega <h:messages /> en la página para ver qué campo está fallando.

package com.veterinaria.controlador;

    @PostConstruct
    public void init() {
        // Datos iniciales de ejemplo (en memoria)
        clientes.add(new Cliente(1, "30123456", "María Gómez"));
        clientes.add(new Cliente(2, "28987654", "Carlos López"));
        clientes.add(new Cliente(3, "35456789", "Ana Martínez"));
        proximoId = 4;
    }

    public void agregar() {
        if (editando) {
            // Ya se modificó directamente el objeto en la lista
            addMsg(FacesMessage.SEVERITY_INFO, "Cliente actualizado correctamente");
            editando = false;
        } else {
            clienteSeleccionado.setId(proximoId++);
            clientes.add(clienteSeleccionado);
            addMsg(FacesMessage.SEVERITY_INFO, "Cliente agregado correctamente");
        }
        clienteSeleccionado = new Cliente();
    }

    public void eliminar(Cliente c) {
        clientes.remove(c);
        addMsg(FacesMessage.SEVERITY_WARN, "Cliente eliminado: " + c.getNombre());
    }

    public void prepararEdicion(Cliente c) {
        clienteSeleccionado = c;
        editando = true;
    }

    public void cancelar() {
        clienteSeleccionado = new Cliente();
        editando = false;
    }

    private void addMsg(FacesMessage.Severity severity, String texto) {
        FacesContext.getCurrentInstance()
            .addMessage(null, new FacesMessage(severity, texto, null));
    }

    // ===== Getters y Setters =====
    public List<Cliente> getClientes()                     { return clientes; }
    public void setClientes(List<Cliente> clientes)        { this.clientes = clientes; }
    public Cliente getClienteSeleccionado()                 { return clienteSeleccionado; }
    public void setClienteSeleccionado(Cliente c)         { this.clienteSeleccionado = c; }
    public boolean isEditando()                           { return editando; }
    public void setEditando(boolean editando)             { this.editando = editando; }
}ClienteBean.java
